# 多媒体互动教材平台 - 产品需求文档

## 1. 产品概述

一个通用型的多媒体互动教材平台，支持任意学科（数学、物理、化学、语文、历史等）的教学内容展示。配备简易后台管理系统，让教师和教育工作者无需编程基础即可管理和发布内容。

- **核心目标**：零门槛内容管理 + 沉浸式多媒体学习体验
- **目标用户**：教师、教育工作者、学生、教育机构

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 访客用户 | 无需注册 | 浏览教材内容、观看动画演示 |
| 教师用户 | 邮箱注册 | 管理自己创建的科目和章节 |
| 管理员 | 系统预设 | 管理所有内容和用户 |

### 2.2 功能模块

#### 前台（学生学习端）
1. **首页**：平台品牌展示、学科分类入口、精选内容推荐
2. **学科页面**：按学科分类展示章节列表
3. **章节详情**：多媒体内容展示（文字、图片、视频、交互元素）
4. **互动演示区**：可配置的参数化交互演示

#### 后台（内容管理端）
1. **仪表盘**：内容概览、快速操作入口
2. **科目管理**：创建、编辑、删除学科分类
3. **章节管理**：创建、编辑、删除章节内容
4. **内容编辑器**：富文本编辑、公式编辑、文件上传

## 3. 核心流程

```
教师登录后台 → 创建/选择科目 → 添加章节 → 编辑章节内容 → 发布
                                    ↓
学生访问首页 → 选择科目 → 浏览章节 → 学习互动内容
```

## 4. 用户界面设计

### 4.1 前台设计风格

- **主色调**：深靛蓝 #1a1a2e 配以渐变紫 #4a0e8f
- **强调色**：霓虹青 #00d4ff，用于交互元素
- **辅助色**：暖橙 #ff6b35，用于进度和成功状态
- **字体**：思源黑体（Noto Sans SC）
- **布局**：卡片式布局，响应式设计

### 4.2 后台设计风格

- **色调**：浅灰背景 #f5f5f5，清新专业
- **强调色**：蓝色 #3b82f6，用于主要按钮
- **布局**：左侧导航 + 右侧内容区
- **编辑器**：类似 Word 的可视化富文本编辑

### 4.3 后台管理功能

| 模块 | 功能 | 说明 |
|------|------|------|
| 科目管理 | 添加/编辑/删除科目 | 设置科目名称、图标、简介 |
| 章节管理 | 添加/编辑/删除章节 | 设置章节标题、排序、发布状态 |
| 内容编辑 | 富文本编辑 | 类似于 Word，所见即所得 |
| 公式插入 | LaTeX 公式 | 一键插入数学/化学公式 |
| 媒体管理 | 图片/视频上传 | 支持拖拽上传 |

### 4.4 响应式设计

- **桌面端**：双栏布局，后台左侧导航240px固定
- **平板端**：后台导航收起为图标模式
- **移动端**：前后台均适配移动端浏览

## 5. 数据模型

### 5.1 科目 (Subject)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| name | VARCHAR(100) | 科目名称，如"高中数学" |
| icon | VARCHAR(50) | 图标，如"∑"、"△" |
| description | TEXT | 科目简介 |
| color | VARCHAR(20) | 主题色 |
| order_index | INTEGER | 排序顺序 |
| created_at | DATETIME | 创建时间 |

### 5.2 章节 (Chapter)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| subject_id | INTEGER | 所属科目ID |
| title | VARCHAR(200) | 章节标题 |
| content | TEXT | 章节内容（HTML格式） |
| formula | TEXT | 配套公式（可选） |
| demo_config | TEXT | 互动演示配置（JSON格式） |
| order_index | INTEGER | 排序顺序 |
| is_published | BOOLEAN | 是否发布 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 5.3 用户 (User)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| email | VARCHAR(100) | 登录邮箱 |
| password | VARCHAR(255) | 密码（加密存储） |
| name | VARCHAR(50) | 显示名称 |
| role | VARCHAR(20) | 角色：admin/teacher |
| created_at | DATETIME | 创建时间 |

## 6. API 设计

| 方法 | 路由 | 说明 |
|------|------|------|
| GET | /api/subjects | 获取所有科目 |
| POST | /api/subjects | 创建科目 |
| PUT | /api/subjects/:id | 更新科目 |
| DELETE | /api/subjects/:id | 删除科目 |
| GET | /api/chapters | 获取章节列表（可按科目筛选） |
| GET | /api/chapters/:id | 获取单个章节 |
| POST | /api/chapters | 创建章节 |
| PUT | /api/chapters/:id | 更新章节 |
| DELETE | /api/chapters/:id | 删除章节 |
| POST | /api/upload | 文件上传 |
